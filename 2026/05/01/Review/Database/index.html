Database

DBS=DB（硬件）+DBMS（软件OS）+USER

E-R图

  • 方框：表示一个实体集
  • 菱形框：表示属性和实体之间的联系（均使用无向边进行连接）
  • 椭圆框：表示属性

数据结构

  • 层次结构：用树状层次结构表示关系（只有根节点没有双亲节点）
    • 采用指针实现，结构简单、层次分明，查询效率高
    • 难以表达非层次的复杂关系，树状结构插入和删除节点困难
  • 网状模型：用有向图表示关系
    • 能够直接表示实体世界的关系
    • 结构、编程复杂
  • 关系模型：用二维表格表示（行：元组）（列：属性）
    • 通过表间的关键字（公共属性）进行联系
  • 面向对象数据模型

系统结构

记录型：(学号，姓名，性别，专业，籍贯，年龄)

记录值：(08241120，李仔仔，男，计算机，江苏南京，21)

两两模式之间有映像，当修改某一个模式时只需要修改映像即可。

  • 外部模式：用户与数据库的接口

    • 外映像：数据逻辑独立性（当模式（全局逻辑结构）改变时，外模式 / 模式映像保证外模式不变，应用程序基于外模式编写，因此应用程序不用修改）

  • 概念模式（模式）：一个数据库只有一个模式

    • 内映像：数据物理独立性（把数据库从机械硬盘迁移到固态硬盘（物理存储改变），应用程序完全感知不到，不用做任何修改）

  • 内部模式：数据库的物理结构和存储方式

  • 选择运算（行）$\sigma$

  • 投影运算（列）$\Pi$

  • 连接运算

    • 直接连接
    • 自然连接（有公共属性的去掉重复列

  • 除法运算

SQL语句

  • 定义主关键字

    • 在属性及其类型后加上保留字PRIMARYKEY
    • 在列出关系模式的所有属性后，附加一个声明PRIMARY KEY (属性)
    • 非空 (NOT NULL) 约束和唯一 (UNIQUE) 约束结合在的作用等同于主键(PRIMARY KEY) 约束

如果主关键字有两个，则只能在属性列表最后来定义关键字。

  • 属性列表末尾进行声明
