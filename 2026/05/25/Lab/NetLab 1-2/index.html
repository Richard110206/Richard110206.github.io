Windows 常用网络命令

以下命令均在 cmd 中运行（Win+R → 输入 cmd → 回车）。

ipconfig — 查看/更新本机网络配置

这是什么？ 查看本机的 IP 地址、子网掩码、网关、DNS 等网络参数。
什么时候用？ 上不了网时第一步就跑这个，看看自己有没有拿到 IP、网关对不对。

参数 含义
/all 显示完整信息（MAC、DHCP、DNS 等）
/release 释放 IP（归还给 DHCP）
/renew 重新获取 IP
/flushdns 清除 DNS 缓存
/displaydns 查看当前 DNS 缓存内容

ping — 测试两台主机能否通信

这是什么？ 向目标发一个 ICMP 包，对方收到后回一个确认包，以此判断网络通不通。
什么时候用？ 上不了网时：先 ping 网关 → 通了说明局域网 OK；再 ping 公网 IP（8.8.8.8）→ 通了说明外网 OK；再 ping 域名 → 不通说明 DNS 有问题。

  • -t 持续 Ping（Ctrl+C 停止）
  • -i TTL 设置生存时间（每经过一个路由器 TTL 减 1，减到 0 则丢弃——这就是网络层数据包的"寿命"）
  • -w timeout 超时等待时间（毫秒）
  • 返回 请求超时 = 对方没响应；返回 目标主机不可达 = 路由不可达

tracert — 追踪数据包经过了哪些路由器

这是什么？ 探测你的电脑到目标之间经过了哪些路由器（每一"跳"）以及每跳的延迟。
什么时候用？ ping 不通时用来定位"断在哪一跳"。比如 ping 不通百度，跑 tracert 看第几跳断了，就能判断问题出在校园网还是运营商。

  • -d 不解析主机名（跳过 DNS 查询，更快）
  • -h maximum_hops 最大跳数（默认 30）
  • 输出中 * * * 表示该跳超时未响应（可能对方禁止了 ICMP）

nslookup — 查询域名对应的 IP 地址

这是什么？ 查询 DNS 记录，即"这个域名对应哪个 IP"。
什么时候用？ 浏览器打不开网站但 ping IP 能通 → 怀疑 DNS 有问题就用这个。

netstat — 查看本机的网络连接状态

这是什么？ 显示本机当前所有网络连接、侦听端口、路由表和协议统计。
什么时候用？ 怀疑电脑有异常程序联网、想看某个端口被谁占用、排查端口冲突。

常用参数：-a 所有端口，-n 数字形式（不解析域名，更快），-o 显示进程 PID，-b 显示程序名（需管理员权限）。

arp — 查看 IP 与 MAC 地址的映射表

这是什么？ ARP（地址解析协议）负责把 IP 地址翻译成网卡物理地址（MAC）。arp -a 查看本机缓存的映射表。
什么时候用？ 局域网通信异常时检查 ARP 表有没有被篡改（ARP 欺骗攻击会改这个表）。

route — 查看和管理本机路由表

这是什么？ 路由表决定你的电脑发出去的数据包该往哪个网关走。
什么时候用？ 多网卡环境（有线+无线+VPN），需要控制数据走哪个出口；或排查"访问某个网段走错了路"。

匹配规则：最长前缀优先。0.0.0.0 是默认路由（兜底），优先级最低。METRIC 越小优先级越高。

netsh — 命令行配置网络参数

这是什么？ Windows 的"网络配置瑞士军刀"，能用命令行完成图形界面里的所有网络配置。
什么时候用？ 批量配置多台电脑、写自动化脚本、图形界面点不开时。

两种用法：通行命令（一行写完）和分级命令（输入 netsh 后进入交互界面逐级选择）。

Wireshark

这是什么？ 全球最流行的开源网络抓包分析工具。它能截获网卡上流经的所有数据包，逐层解析出每一层的协议头部信息（从以太网帧 → IP → TCP → HTTP 应用层），让你"看到"网络中到底在传什么数据。
什么时候用？ 前面那些命令（ping、tracert、netstat）只能看到"结果"（通不通、端口有没有开），而 Wireshark 能看到"过程"——每一个包的完整内容。排错时用命令定位大致问题，再用 Wireshark 精确分析包内容。

过滤器

Wireshark 有两种过滤器：

捕捉过滤器 显示过滤器
作用时机 开始捕捉前设置 捕捉前或捕捉后均可
功能 第一层过滤，控制捕捉数据数量 在已捕捉结果中精细筛选
灵活性 设置后不可修改 随时可修改
复杂度 较简单 更强大、更复杂

捕捉过滤器语法

  • Protocol（协议）： ether, fddi, ip, arp, rarp, tcp, udp 等，未指定则匹配全部协议。关键字作值时需加 \ 转义。

  • Direction（方向）： src, dst, src and dst, src or dst，默认 src or dst。

  • Host(s)（主机标识）： net（网段）, port（单端口）, host（主机）, portrange（端口区间），默认 host。

  • Logical Operations（逻辑运算）： not > and = or（同级从左向右）。

  • 特殊标识： 仅可搭配 ip、ether 使用 —— multicast（组播）, broadcast（广播）, no broadcast（排除广播）。

示例：tcp dst 10.1.1.1 80 and tcp dst 10.2.2.2 3128

显示过滤器语法

比较运算符：

英文 C 语言 含义
eq == 等于
ne != 不等于
gt > 大于
lt < 小于
ge >= 大于等于
le <= 小于等于

逻辑运算符：

英文 C 语言 含义
and && 逻辑与
or || 逻辑或
xor ^^ 逻辑异或（仅一个满足时为真）
not ! 逻辑非

输入正确时背景变绿色，错误时变红色。

OSI 层级与抓包信息的关系

  • OSI Layer 2 抓包：Source/Destination 列显示 MAC 地址，Port 列为空
  • OSI Layer 3 及以上：Source/Destination 列显示 IP 地址
  • Layer 4 及以上：Port 列显示 TCP/UDP 端口号

Cisco Packet Tracer (PT)

这是什么？ Cisco 开发的免费网络仿真程序，让你在电脑上虚拟搭建路由器、交换机、PC 等设备，组成一个完整的网络拓扑。它可以模拟数据包在网络中逐跳转发的全过程。
什么时候用？ 真实设备太贵不方便，PT 提供了一个安全的实验环境。学完网络理论后，用 PT 搭一个网络验证理论（比如配 RIP 路由、划 VLAN、配 NAT），比看书直观得多。

两种工作状态

逻辑状态（Logical） 物理状态（Physical）
用途 网络设备的逻辑连接及配置（拓扑设计、设备选型、端口参数配置） 地理布局设计（城市、办公室等地理位置关系）

两种工作模式

实时模式（Realtime） 模拟模式（Simulation）
特点 连续时间，模拟实际网络工作 时间暂停，人工控制逐跳传输
用途 设备配置、IOS CLI 模拟、即时连通性测试 可视化观察数据包逐跳转发过程、排错

连接线类型

线型 用途
自动选线 不确定时使用，一般不建议
直通线 不同设备相连（“异直”）
交叉线 相同设备相连，或路由器↔PC（“同反”）
光纤 光纤连接
DCE/DTE 路由器之间串行连接，DCE 端需配置时钟

原则：“同反异直” —— 相同设备交叉线，不同设备直通线。

线缆端点状态指示

圆点状态 含义
亮绿色 物理连接就绪，尚未有 Line Protocol status 指示
闪烁绿色 连接已激活
红色 物理连接不通，没有信号
黄色 交换机端口处于"阻塞"状态

设备配置界面

设备的配置界面有三个标签：

  • Physical — 模块的物理组成配置（添加/删除模块前必须关闭电源，不支持热插拔）
  • Config — GUI 图形化配置，包含 GLOBAL / ROUTING / SWITCHING / INTERFACE 四个大项
  • CLI — 命令行配置，与实际路由器配置环境相似

常见排错要点

  • 路由器默认端口有限，连接前需确认端口是否够用，必要时在 Physical 标签下添加模块（如 NM-4E）
  • 添加/删除模块前必须关闭设备电源
  • PC 需正确配置 IP 地址、子网掩码、默认网关
  • 网段划分错误或漏配默认网关会导致 Ping 不通
  • 组网遵循 “配置 - 测试 - 排错” 的循环
