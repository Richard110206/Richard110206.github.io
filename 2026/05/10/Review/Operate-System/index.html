Operate System

第一章 概论

并发与并行

  • 并行：多个事件同一时刻发生（需要多个 CPU）
  • 并发：多个事件同一时间段内发生（宏观同时，微观交替）

OS 四大特征

并发性、共享性、虚拟性、随机性（异步性）

Q：OS 的四大特征分别怎么理解？

  • 并发性：多个程序宏观上同时运行（最基本特征，是 OS 存在的根本原因）
  • 共享性：系统资源被多个进程共用（互斥共享 or 同时共享）
  • 虚拟性：把一个物理实体变为多个逻辑对应（如虚拟内存、虚拟 CPU）
  • 随机性（异步性）：程序执行顺序和时机不确定（如中断发生时机不可预知）

并发和共享是 OS 两个最基本的特征，二者互为存在条件。

OS 的定义与功能

  • 定义：操作系统是控制和管理整个计算机系统的硬件和软件资源，合理地组织、调度计算机的工作与资源的分配，进而为用户和其他软件提供方便接口与环境的最基本的系统软件。
  • 功能：
    • 进程管理（处理机管理）
    • 存储管理（内存管理）
    • 设备管理
    • 文件管理
    • 网络通信与服务
    • 安全与保护

OS 发展

阶段 特点
联机批处理 作业通过 I/O 设备直接提交给主机，CPU 与 I/O 串行
脱机批处理 输入输出通过卫星机完成，减少 CPU 等待 I/O
多道批处理 内存中同时放多道程序，宏观并行微观串行，资源利用率高但无交互
分时 OS 多用户共享一台主机，每个用户一个终端，人机交互、独占感，采用时间片轮转
实时 OS 要求在严格时间限制内响应，分为硬实时和软实时

Q：多道批处理 vs 分时 OS？

  • 多道批处理：追求吞吐量和资源利用率，用户不能交互，无人为干预
  • 分时 OS：追求响应速度，用户可以交互，采用时间片轮转，具有多路性、交互性、独立性、及时性

关键区别：多道批处理无交互能力；分时系统有人机交互能力。

OS 接口

  • 程序接口（系统调用/API）：程序通过软中断（trap）请求 OS 服务，是用户程序请求 OS 服务的唯一方式。包括：设备管理、文件管理、进程控制、进程通信、内存管理
  • 操作接口：命令行（shell）、GUI（图形界面）

Q：用户程序如何使用 OS 的功能？
通过系统调用（程序接口）。用户程序在用户态运行，需要 OS 服务时触发软中断（trap），切换到核心态执行 OS 代码，完成后返回用户态。

注意：系统调用是应用程序与 OS 之间的接口；库函数是编程语言提供的函数（可能封装了系统调用）。

【2024 春真题】操作系统提供给用户的接口有哪几种？

  • 命令接口：联机命令接口（交互式）、脱机命令接口（批处理）
  • 程序接口：系统调用（广义指令）
  • 图形接口：GUI

第二章 进程管理

程序的运行方式

方式 特点
顺序运行 一个程序执行完才执行下一个，具有顺序性、封闭性、可再现性
并发运行 多个程序宏观上同时运行，具有间断性、失去封闭性、不可再现性
并行运行 多个程序在同一时刻真正同时运行（需要多核/多 CPU）

进程与线程

  • 进程：资源分配的基本单位，拥有独立地址空间
  • 线程：CPU 调度的基本单位，同一进程的线程共享地址空间
  • PCB（进程控制块）：OS 感知进程存在的唯一标志，包含进程标识、处理机状态、进程调度信息、进程控制信息

Q：进程、程序、作业的区别？

  • 程序：静态的，存放在磁盘上的代码和数据（被动实体）
  • 进程：动态的，程序的一次执行过程（有生命周期：创建→执行→撤销）
  • 作业：用户要求计算机完成的任务，从提交到完成的全过程（包含多个进程）
  • 关系：作业 → 调入内存成为进程 → 进程执行的是程序

进程状态

  • 就绪→运行：被调度程序选中
  • 运行→就绪：时间片到 / 被抢占
  • 运行→阻塞：等待 I/O、申请资源（主动行为）
  • 阻塞→就绪：I/O 完成、资源可用（被动行为，需要别的来唤醒）

做题注意：没有 阻塞→运行 的直接转换；进程可以自己阻塞自己，但不能自己唤醒自己

挂起状态：进程被换出到外存。

  • 就绪→就绪挂起（挂起）
  • 阻塞→阻塞挂起（挂起）
  • 就绪挂起→就绪（激活）
  • 阻塞挂起→就绪挂起（等待的事件完成）

三级调度

调度层次 功能 频率
高级调度（作业调度） 外存→内存，决定哪个作业调入 最低
中级调度（内存调度） 内外存交换，涉及挂起/激活 中等
低级调度（进程调度） 就绪队列→CPU，分配处理机 最高

进程调度算法

算法 思路 特点
FCFS（先来先服务） 按到达顺序 非抢占，简单，对短作业不利，不会饥饿
SJF/SPF（短作业优先） 选执行时间最短的 非抢占，平均等待时间最短，可能饥饿
SRTF（最短剩余时间优先） SJF 的抢占版 新进程到达时比较剩余时间，可能饥饿
HRRN（高响应比优先） $RP = \frac{等待时间 + 要求服务时间}{要求服务时间}$ 兼顾长短作业，不可抢占，不会饥饿
RR（时间片轮转） 就绪队列轮转执行 时间片太大退化为 FCFS，太小切换开销大

做题关键指标：

  • 周转时间 = 完成时间 - 到达时间
  • 带权周转时间 = 周转时间 / 服务时间
  • 平均周转时间 = 所有进程周转时间之和 / 进程数
  • 平均带权周转时间 = 所有进程带权周转时间之和 / 进程数

做题方法：画甘特图，标注每个时刻谁在运行。SRTF 抢占式：每个新进程到达时刻都要重新比较剩余时间。

两道作业批处理系统：作业调度用 SJF（内存有空位时选最短作业调入），进程调度用优先权抢占（内存中的作业按优先级执行）。注意内存容量限制——只有内存有空位时才触发作业调度。

作业调度采用 SJF，进程调度采用优先数抢占（优先数越小优先级越高），两道作业：

【作业题】SPF 与 SRTF 对比

4 个进程：P1(到达 0, CPU 10)、P2(到达 2, CPU 4)、P3(到达 4, CPU 10)、P4(到达 6, CPU 5)

SPF（非抢占）：P1(0-10) → P2(10-14) → P4(14-19) → P3(19-29)

  • 平均周转时间 = (10+12+25+13)/4 = 15
  • 平均带权周转时间 = (1+3+2.5+2.6)/4 = 2.275

SRTF（抢占）：P1(0-2) → P2(2-6) → P4(6-11) → P1(11-19) → P3(19-29)

  • 平均周转时间 = (19+4+25+5)/4 = 13.25
  • 平均带权周转时间 = (1.9+1+2.5+1)/4 = 1.6

进程同步与互斥

临界资源：一段时间内只允许一个进程使用的资源（如打印机、共享变量）

临界区：访问临界资源的代码段。进入区 → 临界区 → 退出区 → 剩余区

同步机制应遵循的准则：

  1. 空闲让进：临界区空闲时，允许进程进入
  2. 忙则等待：已有进程进入临界区时，其他进程必须等待
  3. 有限等待：进程等待进入临界区的时间应有限（不饥饿）
  4. 让权等待：不能进入临界区的进程应释放 CPU

信号量（PV 操作）

信号量 $S$ 表示资源数量：

  • P(S)：$S = S - 1$；若 $S < 0$ 则阻塞等待（进程放入等待队列）
  • V(S)：$S = S + 1$；若 $S \leq 0$ 则唤醒一个等待进程

$S$ 的值 = 可用资源数；$S < 0$ 时 $|S|$ = 等待队列中的进程数

P 操作：申请资源，可能阻塞；V 操作：释放资源，不会阻塞

哲学家进餐

5 个哲学家围坐圆桌，每两人之间放一根筷子，哲学家需同时拿到左右两根筷子才能进餐。若每人同时拿起左边筷子，则形成循环等待 → 死锁。

方法一：限制同时进餐人数（最多允许 4 人同时尝试拿筷子，破坏循环等待）

方法二：奇偶策略（破坏循环等待——奇数号先拿左、偶数号先拿右）

方法三：仅当能同时拿到两根筷子时才拿（用互斥信号量保护取筷子操作）

三种方法对比：

  • 方法一：简单，但并发度受限（最多 4 人同时尝试）
  • 方法二：不打断拿筷子过程，并发度更高
  • 方法三：最直观，但 mutex 会使得同一时刻只有一人能尝试拿筷子，并发度最低

死锁

产生条件（必须同时满足四个）

  1. 互斥：资源独占
  2. 不可剥夺（不剥夺）：已获得的资源不能被强行收回
  3. 请求并保持（占有并等待）：持有资源的同时请求新资源
  4. 循环等待（环路等待）：进程间形成环形等待链

Q：死锁 vs 饥饿 vs 活锁？

  • 死锁：进程互相等待，永久阻塞
  • 饥饿：进程长期得不到服务（如短作业优先下长作业等不到）
  • 活锁：进程不断重试但始终无法成功

处理策略

策略 方法 特点
预防 破坏四个条件之一 如：顺序分配资源破坏循环等待；一次性申请所有资源破坏请求并保持
避免 银行家算法 动态检查资源分配是否安全
检测+解除 资源分配图检测环 剥夺资源/撤销进程解除

银行家算法

数据结构：

  • Available（系统可用资源向量）
  • Max（各进程最大需求矩阵）
  • Allocation（各进程已分配矩阵）
  • Need = Max - Allocation（各进程剩余需求矩阵）

安全性算法：找 Need ≤ Available 的进程，分配后回收资源，重复直到所有进程完成（安全序列）。

做题步骤：

  1. 写出 Available 向量
  2. 逐行检查 Need ≤ Available（逐分量比较，都要满足）
  3. 找到满足的进程 → 标记完成 → 回收其 Allocation（Available += Allocation）
  4. 重复直到所有进程完成（安全）或找不到可满足的进程（不安全）

请求分配判定：

  1. 检查 Request ≤ Need（是否超过最大需求）
  2. 检查 Request ≤ Available（系统是否有足够资源）
  3. 试分配：Allocation += Request，Available -= Request，Need -= Request
  4. 安全性检查：安全则分配，不安全则等待（恢复原状态）

【作业题】银行家算法例题

5 个进程 P1~P5，3 类资源 A B C，T0 时刻状态：

Available = (2, 3, 3)

进程MaxAllocationNeed
P1(5,5,9)(2,1,2)(3,4,7)
P2(5,3,6)(4,0,2)(1,3,4)
P3(4,0,11)(4,0,5)(0,0,6)
P4(4,2,5)(2,0,4)(2,2,1)
P5(4,2,4)(3,1,4)(1,1,0)

T0 安全性检查：

  • P5(1,1,0) ≤ (2,3,3) ✓ → 回收后 Available=(5,4,7)
  • P4(2,2,1) ≤ (5,4,7) ✓ → 回收后 Available=(7,4,11)
  • P1(3,4,7) ≤ (7,4,11) ✓ → 回收后 Available=(9,5,13)
  • P2(1,3,4) ≤ (9,5,13) ✓ → 回收后 Available=(13,5,15)
  • P3(0,0,6) ≤ (13,5,15) ✓

  • (2,0,1) ≤ Need(2,2,1) ✓，(2,0,1) ≤ Available(2,3,3) ✓

第三章 内存管理

地址重定位

将逻辑地址转换为物理地址：

  • 静态重定位：装入时一次性转换，装入后不能移动（无需硬件支持）
  • 动态重定位：运行时动态转换（靠重定位寄存器），程序可以移动（需硬件支持）

覆盖与交换

技术 机制 特点
覆盖 程序级的内存扩展，按段调入 用户需手动声明覆盖结构，对用户不透明
交换 系统级的，整个进程换入换出 以进程为单位，由 OS 自动管理，对用户透明

分区管理

算法 策略 特点
最先适应（首次适应） 第一个够大的分区 简单，低地址碎片多，最常用
循环首次适应 从上次位置继续找 碎片分布均匀
最优适应 最小的够用分区 产生很多微小碎片（外部碎片）
最坏适应 最大的分区 避免微小碎片，但大作业可能分配不到
快速适应 按大小分类的空闲链 不会产生外部碎片，但回收困难

碎片问题：

  • 内部碎片：分配给进程的内存大于实际需要（分区内部的浪费），页式存储有内部碎片
  • 外部碎片：内存中空闲空间太小无法利用（分区之间的浪费），分区式有外部碎片
  • 紧凑（compaction）可以解决外部碎片，但需要动态重定位支持

页式存储

分区式管理是连续分配，会产生碎片。页式将逻辑地址映射到离散的物理地址（靠页表让进程以为自己是连续的）。

页面大小为 $L$，逻辑地址为 $A$：

$$P = \lfloor A / L \rfloor, \quad d = A \bmod L$$

物理地址 = 块号 × 块大小 + 页内偏移 $d$

做题关键：页内偏移量 = 页面大小对应的位数 = 页内地址。页表项数 = 逻辑地址空间所占的页数

十六进制地址转换步骤：

  1. 将十六进制转二进制
  2. 按地址结构划分：页号位数 + 页内偏移位数
  3. 页号查页表得块号，偏移量不变
  4. 物理地址 = 块号拼接偏移量（或 块号×页面大小 + 偏移量）

【作业题】逻辑地址转换例题

用户空间 32 页，每页 1KB，物理内存 16KB（16 块），页表：页 0→块 5、页 1→块 10、页 2→块 4、页 3→块 7。

逻辑地址 0A5CH：

  1. 地址结构：32 页 → 页号 5 位；1KB 页 → 偏移 10 位；共 15 位
  2. 0A5CH = 0000 1010 0101 1100B，取低 15 位：000 1010 0101 1100
  3. 页号 = 00010B = 2，偏移 = 10 0101 1100B = 25CH
  4. 查页表：页 2 → 块 4
  5. 物理地址 = 4 × 1KB + 25CH = 1000H + 25CH = 125CH

多级页表：

  • 二级页表访问内存需要 3 次访问（页目录 → 页表页 → 实际数据）
  • 引入目的：减少页表占用的内存空间（不需要连续存放整个页表）

【作业题】二级页表结构

32 位地址，顶级页表 9 位，二级页表 11 位：

  • 页内偏移 = 32 - 9 - 11 = 12 位，页面大小 = $2^{12}$ = 4KB
  • 页号总位数 = 9 + 11 = 20 位，页面总数 = $2^{20}$ = 1M 个页面

快表（TLB）：高速缓存页表项，命中时只需 1 次内存访问

段式存储

按逻辑功能分段（如代码段、数据段、栈段），每段有自己的段号和段内偏移。

段表项：段号 → 基址 + 段长

做题时注意：访问时必须检查段内偏移是否越界（偏移 < 段长），不满足则产生地址越界中断

分页 vs 分段：

对比项 分页 分段
划分依据 系统行为（页大小固定） 用户可见（段大小不固定、按逻辑划分）
地址空间 一维（只需一个地址） 二维（段号+段内偏移）
碎片 有内部碎片，无外部碎片 有外部碎片，无内部碎片
信息共享 不方便 方便（按逻辑段共享）

段页式存储

先分段，段内再分页。逻辑地址 = 段号 + 页号 + 页内偏移

访问过程：查段表 → 查页表 → 访问内存（共 3 次内存访问）

虚拟存储器

基于局部性原理（时间局部性、空间局部性），只装入部分页面/段，缺页时再从外存调入。

  • 虚拟存储器最大容量 = min(内存+外存, 地址结构能表示的最大容量)
  • 缺页率 $f = \frac{\text{缺页次数}}{\text{总访问次数}}$

Q：虚拟存储器实现方式？

  • 请求分页存储管理（最常用）
  • 请求分段存储管理
  • 请求段页式存储管理

虚拟存储器特征：多次性（分多次调入）、对换性（换入换出）、虚拟性（逻辑上扩充内存）

页面调入策略与分配策略

Q：页面调入策略与分配策略怎么考？

  • 调入时机：请求调页（缺页时才调入，常用）vs 预调页（提前调入，减少缺页）
  • 调入来源：从对换区调入（快，I/O 效率高）vs 从文件系统调入（慢）
  • 分配策略：固定分配（每个进程页框数固定）vs 可变分配（动态调整）
  • 置换范围：局部置换（只在自己的页框中选）vs 全局置换（可从其他进程抢页框）
  • 常考搭配：固定分配+局部置换 / 可变分配+全局置换

页面置换算法

算法 淘汰策略 特点
OPT 淘汰以后最长时间不用的 理论最优，无法实现（需预知未来）
FIFO 淘汰最早进入的 简单，有 Belady 异常（帧数增加缺页反而增多）
LRU 淘汰最近最久未使用的 效果接近 OPT，实现开销大，无 Belady 异常
Clock 近似 LRU，用引用位环形扫描 实现简单，效果适中

做题技巧：

  1. 画表格，每列一个页面号，逐个时刻填入帧中页面
  2. 命中打 ✓，缺页打 ✗
  3. 初始状态注意已装入的页（如第 0 页已装入，第 0 页不算缺页）
  4. FIFO 注意 Belady 异常
  5. Clock 算法：指针循环扫描，访问位为 0 则淘汰，为 1 则置 0 继续扫

缺页率 = 缺页次数 / 总访问次数

第四章 设备管理

设备分类

分类方式 类型
按使用特征 独占设备（打印机，全程独占）、共享设备（磁盘，可交替用）、虚拟设备（SPOOLing 模拟的共享设备）
按信息交换单位 块设备（磁盘，以块为单位，可寻址）、字符设备（键盘/打印机，以字符为单位，不可寻址）

Q：独占设备、共享设备、虚拟设备的区别？

  • 独占设备：一旦分配给某进程，其他进程不能使用（如打印机、磁带机）
  • 共享设备：多个进程可交替使用，可寻址（如磁盘）
  • 虚拟设备：通过 SPOOLing 技术把独占设备改造为多进程共享的逻辑设备

I/O 控制方式

方式 CPU 干预 数据单位 特点
程序直接控制（轮询） 每次都要轮询 字 CPU 忙等，效率最低
中断驱动 I/O 完成后中断 CPU 字 CPU 可做其他事，但每字一次中断
DMA 传完一块才中断 块 减少 CPU 干预和中断次数
通道 通道独立完成 I/O 一组块 CPU 几乎不参与，通道是专用处理器

注意层次：程序 I/O < 中断 < DMA < 通道，CPU 参与度递减，效率递增。

DMA 与通道的区别：

  • DMA 只能传数据，通道有自己的指令（通道程序），功能更强
  • DMA 一块一中断，通道一组块一中断

I/O 软件层次（从上到下）

  1. 用户层 I/O 软件
  2. 设备无关软件（设备独立性软件）
  3. 设备驱动程序
  4. 中断处理程序

SPOOLing 技术

假脱机技术（Simultaneous Peripheral Operations On-Line）：

  • 用磁盘作为缓冲（输入井/输出井），将独占设备虚拟为共享设备
  • 组成：输入井 + 输出井 + 输入缓冲区 + 输出缓冲区 + 输入进程 SPI + 输出进程 SPO
  • 典型应用：共享打印机（多个进程同时打印，先缓存在输出井，逐个打印）

Q：SPOOLing 系统的组成与工作原理？

  • 输入井：磁盘上的输入缓冲区，暂存 I/O 设备输入的数据
  • 输出井：磁盘上的输出缓冲区，暂存用户程序的输出数据
  • 输入缓冲区：内存中，暂存从输入设备到输入井的数据
  • 输出缓冲区：内存中，暂存从输出井到输出设备的数据
  • 输入进程：将 I/O 设备数据读入输入井
  • 输出进程：将输出井数据送到输出设备

磁盘调度

磁盘访问时间 = 寻道时间 + 旋转延迟 + 传输时间

寻道时间是主要优化目标（占总时间比重最大）

算法 策略 特点
FCFS 按请求顺序 公平但寻道长
SSTF（最短寻道时间优先） 最近磁道优先 局部最优，可能饥饿
SCAN（电梯算法） 单方向扫到头再反向 无饥饿
C-SCAN 只在单方向服务，到头直接返回起点 更均匀的等待时间
LOOK / C-LOOK 到最远请求即返回，不到物理端点 实际更常用

做题步骤：画出磁道编号轴 → 标记当前磁头位置和请求序列 → 按算法画移动路径 → 计算总移动距离

注意：

  • SCAN 到物理端点；LOOK 到最远请求就回头
  • C-SCAN 反方向不服务；SCAN 反方向也服务

第五章 文件系统

文件逻辑结构

结构 特点
流式结构（无结构） 字节序列，如 txt 文件
记录式结构 顺序式 / 索引式 / 索引顺序式

文件访问方式

方式 特点
顺序访问 按记录顺序依次读写，适合磁带
直接访问（随机访问） 直接跳到指定位置读写，适合磁盘
索引访问 先查索引表找到位置，再直接访问

Q：文件访问方式和物理结构怎么对应？

  • 连续分配 → 支持顺序访问和直接访问
  • 链接分配 → 主要支持顺序访问，直接访问效率低
  • 索引分配 → 支持直接访问和顺序访问

文件物理结构

结构 优点 缺点
连续分配 顺序读写快，支持直接访问 外部碎片，难以扩展，需要预知文件大小
链接分配（隐式） 无外部碎片，文件可动态扩展 不支持高效直接访问，指针占空间
链接分配（显式/FAT） 支持直接访问（查 FAT 表） FAT 表占内存
索引分配 支持直接访问，无外部碎片 小文件浪费索引块，大文件需要多级索引

混合索引计算（重点）

Unix 方式：直接地址 + 一次间接 + 二次间接 + 三次间接

做题关键参数：

  • 磁盘块大小 = 物理块大小 = 簇大小（如 1024B = 1KB）
  • 地址项长度 = 盘块号大小（如 4B）
  • 每个索引块可存的块号数 = 磁盘块大小 / 地址项长度 = 1024/4 = 256 个

计算公式：

  • 直接地址块：直接指向数据块（如 10 个直接块）
  • 一次间接：可寻址 256 块
  • 二次间接：可寻址 $256^2 = 64K$ 块
  • 三次间接：可寻址 $256^3 = 16M$ 块
  • 最大文件长度 = 总块数 × 磁盘块大小

例：10 个直接块 + 1 个一次间接 + 1 个二次间接 + 1 个三次间接

  • 最大 = (10 + 256 + 64K + 16M) × 1KB

目录结构

结构 特点
单级目录 所有文件在同一目录，不能重名
两级目录 按用户分（MFD + UFD），不同用户可重名
树形目录 层次结构，路径唯一标识，最常用
无环图目录 支持文件共享

FCB（文件控制块）：存放文件元信息（文件名、物理地址、文件大小、创建时间等），是目录项的核心。FCB 的有序集合称为文件目录。

索引节点（inode）：将文件名与文件描述信息分开存储，目录项只存文件名和指向 inode 的指针，加快目录检索速度。

文件存储空间管理

方法 原理 特点
位示图（bitmap） 每位对应一个盘块，1=已分配，0=空闲 空间开销小，适合各种分配
空闲区表法 记录空闲区的起始位置和长度 适合连续分配
空闲块链表法 将空闲块用链表串起来 分配和回收简单
成组链接法 空闲块分组管理（Unix 使用） 高效管理大量空闲块

磁盘容量计算

磁盘总容量 = 柱面数 × 磁道数/柱面(=盘面数) × 扇区数/磁道 × 字节数/扇区

磁盘地址结构：柱面号 + 盘面号(磁头号) + 扇区号

注意区分：

  • 柱面号：确定在哪个柱面（磁道位置）
  • 盘面号（磁头号）：确定在哪个盘面
  • 扇区号：确定在磁道上的哪个扇区

文件共享与保护

  • 硬链接：多个目录项指向同一个 inode（索引节点），计数器记录链接数
  • 软链接（符号链接）：创建一个新文件，内容是指向原文件的路径

硬链接 vs 软链接：

  • 硬链接：删除原文件不影响其他链接（计数器-1）；不能跨文件系统
  • 软链接：原文件删除后链接失效（悬空链接）；可以跨文件系统
